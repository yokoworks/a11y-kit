# @yokoworks/a11y-kit

A lightweight, accessibility-first UI kit built on top of Radix Primitives.
Focus: **practical defaults**, **visible focus ring (cannot be disabled)**, and **clear responsibility boundaries**.

---

## Design Tokens (minimum)

- `color.{primary|neutral|success|warning|danger}.{base|hover|fg}`
- `space.{xs|sm|md|lg|xl}`
- `radius.{sm|md|lg}`
- `ring.{color|width|offset}`
- `font.{family|size.base|line-height.base}`

To use tokens:

```ts
import '@yokoworks/a11y-kit/theme';
```

---

## Common API (for most components)

| Prop        | Type                                | Default   | Notes                                  |
| ----------- | ----------------------------------- | --------- | -------------------------------------- |
| `variant`   | `'default' \| 'outline' \| 'ghost'` | `default` | Visual style                           |
| `size`      | `'sm' \| 'md' \| 'lg'`              | `md`      | Size scale                             |
| `disabled`  | `boolean`                           | `false`   | Native disabled                        |
| `className` | `string`                            | `-`       | Extend/override classes                |
| `asChild`   | `boolean`                           | `false`   | Radix asChild passthrough (when exist) |

---

## Accessibility Guardrails (library policy)

- **Focus ring is always visible** (`:focus-visible`) and **cannot be disabled**.
  Color/width/offset are customizable via tokens.
- **Icon-only buttons require an accessible name**.
  Missing name triggers a dev-time warning.
- **Dialog**: initial focus target and return focus are **required** (API or documented pattern).
- **Forms**: error/help text must be linked via `aria-describedby`; helper utilities provided.

---

## Responsibility Boundaries (per component)

### Dialog

- **Radix:** Focus trap, ARIA roles, portal
- **Library:** Enforced visible focus ring, API for focus return
- **You:** Choose initial focus, provide labels/descriptions

### DropdownMenu

- **Radix:** Roving tabindex, keyboard navigation, roles
- **Library:** Visible focus ring, minimal styling
- **You:** Provide item labels, handle actions

### Tooltip

- **Radix:** Positioning, show/hide semantics
- **Library:** Delay defaults, ring consistency
- **You:** Use for supplemental info only

### Input

- **Radix:** Native semantics
- **Library:** Error/Help components & linking guidance
- **You:** Link via `aria-describedby`

### FormField

- ✅ Exactly **one control child** (guarded in dev)
- ✅ Non-empty visible **label** required
- ✅ `aria-describedby` automatically concatenates **helper → error**  
  → Screen readers will read “help text… then error message”
